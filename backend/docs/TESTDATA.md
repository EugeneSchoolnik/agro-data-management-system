## Аналіз структури БД

### Таблиці бази даних

```
┌─────────────────┐
│     crops       │  - Типи культур (пшениця, кукурудза тощо)
└────────┬────────┘
         │ 1:N
         │
┌────────▼────────┐      ┌──────────────────┐
│     fields      │──┬──→│     sensors      │  - Датчики на полі
└─────────────────┘  │   └────────┬─────────┘
                     │            │ 1:N
                     │   ┌────────▼──────────┐
                     │   │     metrics      │  - Сирі дані датчиків
                     │   └──────────────────┘
                     │
                     │
                     ├──→ ┌──────────────┐
                     │    │   pests      │  - Типи шкідників
                     │    └──────┬───────┘
                     │           │ N:N
                     │   ┌───────▼────────┐
                     └──→│   forecasts    │  - Прогнози ризиків
                         └────────────────┘
```

### Опис таблиць

#### 1. **crops** (культури)

- `id` - унікальний идентифікатор
- `name` - назва культури
- `variety` - сорт
- `description` - опис

#### 2. **fields** (поля)

- `id` - унікальний идентифікатор
- `name` - назва поля
- `area` - площа в гектарах
- `location` - географічне розташування
- `crop_id` - посилання на культуру
- `created_at` - дата створення

#### 3. **sensors** (датчики)

- `id` - унікальний идентифікатор
- `field_id` - посилання на поле
- `sensor_type` - тип датчика (temperature, humidity, soil_moisture)
- `status` - статус (active, inactive, error, testing)
- `last_sync` - час останнього синхронізування

#### 4. **metrics** (метрики)

- `id` - унікальний идентифікатор
- `sensor_id` - посилання на датчик
- `value` - значення вимірювання
- `recorded_at` - час запису

#### 5. **pests** (шкідники)

- `id` - унікальний идентифікатор
- `name` - назва шкідника (українська)
- `scientific_name` - наукова назва

#### 6. **forecasts** (прогнози)

- `id` - унікальний идентифікатор
- `field_id` - посилання на поле
- `pest_id` - посилання на шкідника
- `probability` - ймовірність появи (0-1)
- `recommendation` - рекомендація
- `created_at` - дата створення
