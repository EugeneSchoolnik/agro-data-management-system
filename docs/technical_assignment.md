# ТЕХНІЧНЕ ЗАВДАННЯ

**на розробку бакалаврської кваліфікаційної роботи**

## 1. ЗАГАЛЬНІ ВІДОМОСТІ

- **Тема роботи:** Розробка веб-системи автоматизованого управління аграрними процесами з елементами прогнозування.

## 2. ПРИЗНАЧЕННЯ ТА МЕТА СИСТЕМИ

- **Призначення:** Система призначена для автоматизації збору агрокліматичних даних та інтелектуального прогнозування ризиків у сільському господарстві.
- **Мета:** Підвищення ефективності управління аграрними процесами шляхом впровадження математичних моделей передбачення стану об'єктів моніторингу на основі нейронних мереж.

## 3. ТЕХНІЧНІ ВИМОГИ ТА СТЕК ТЕХНОЛОГІЙ

Система будується за мікросервісною архітектурою:

- **Backend API:** Мова програмування **Golang** (фреймворк **Gin**) для обробки запитів та керування бізнес-логікою.
- **Forecasting Engine:** Мова програмування **Python** для реалізації аналітичних модулів та моделей штучного інтелекту.
- **Database:** Реляційна СУБД **PostgreSQL** для збереження метрик датчиків та результатів прогнозів.
- **Frontend:** Фреймворк **Svelte** для реалізації реактивного інтерфейсу користувача.

## 4. ФУНКЦІОНАЛЬНІ ВИМОГИ

Програмний комплекс має забезпечувати виконання наступних функцій:

### 4.1. Модуль моніторингу

- Автоматизований збір даних щодо стану біоматеріалу та навколишнього середовища.
- Візуалізація результатів аналізу якості середовища за допомогою технологій комп'ютерного зору.

### 4.2. Модуль прогнозування

- Передбачення пошкоджень озимої пшениці клопом-черепашкою (_E. integriceps_) за допомогою нейронних мереж.
- Прогнозування динаміки чисельності шкідників польових культур на основі інтелектуального аналізу даних.
- Формування рекомендацій щодо управління процесами на основі отриманих прогнозів.

### 4.3. Адміністрування

- Керування параметрами об'єктів моніторингу (поля, датчики, типи культур).
- Налаштування системи автоматичних сповіщень про критичні ризики.

## 5. МАТЕМАТИЧНЕ ЗАБЕЗПЕЧЕННЯ

В системі реалізуються наступні математичні підходи:

- **Штучні нейронні мережі:** Використання багатошарових архітектур для прогнозування біологічних загроз.
- **Алгоритми обробки зображень:** Застосування комп'ютерного зору для швидкого оцінювання якості матеріалів.

## 6. ВИМОГИ ДО ІНТЕРФЕЙСУ

- Реалізація у вигляді Single Page Application (SPA).
- Наявність інтерактивних дашбордів з графіками показників у часі.
- Підтримка візуалізації просторових даних про стан угідь.
